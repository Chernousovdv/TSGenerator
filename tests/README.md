# Тестирование латентных компонент генератора синтетических временных рядов

## Обзор

Модуль тестирования предоставляет инструменты для:
- **Базового тестирования** латентных компонент (ARIMA, KernelSynth, TSI, ETS)
- **Профилирования времени** выполнения каждой компоненты
- **Расширенных статистик** для анализа характеристик рядов (17 метрик)
- **Визуализации** результатов и анализа чувствительности к параметрам
- **Анализа реальных датасетов** из Monash Time Series Repository (TSF формат)

## Быстрый старт

```bash
cd ts_generator

# Полный набор тестов (базовое + профилирование + статистики)
python tests/test_latent_components.py --mode full

# Только базовое тестирование + визуализация чувствительности
python tests/test_latent_components.py --mode basic

# Только профилирование времени выполнения
python tests/test_latent_components.py --mode profile

# Анализ реального датасета (быстрый режим, 6 статистик)
python tests/test_latent_components.py --mode real \
    --tsf-path real_datasets/cif_2016_dataset.tsf --fast

# С кастомными параметрами
python tests/test_latent_components.py --mode full --T 200 --samples 20 --device cpu
```

## Аргументы командной строки

### Для режимов `basic`, `full`, `profile`:

| Аргумент | По умолчанию | Описание |
|----------|--------------|----------|
| `--mode` | `full` | Режим тестирования: `basic`, `full`, `profile` или `real` |
| `--T` | `100` | Длина временного ряда |
| `--samples` | `10` | Число сэмплов для статистик |
| `--device` | `cpu` | Устройство: `cpu` или `cuda` |
| `--save-dir` | `./test_reports` | Директория для результатов |

### Для режима `real` (анализ реальных датасетов):

| Аргумент | По умолчанию | Описание |
|----------|--------------|----------|
| `--mode` | `real` | Режим анализа реальных данных |
| `--tsf-path` | - | Путь к TSF файлу (обязательно) |
| `--category` | None | Имя категории (по умолчанию - имя файла) |
| `--max-series` | 100 | Максимум рядов для обработки |
| `--fast` | False | Быстрый режим (только 6 статистик для гистограмм) |
| `--save-dir` | `./test_reports` | Директория для результатов |

## Режимы работы

### Режим `real` - Анализ реальных датасетов

Предназначен для вычисления статистик и визуализации распределений реальных временных рядов из Monash Repository.

**Быстрый режим (--fast):**
- Вычисляет 6 базовых статистик: mean, std, acf_lag1, perm_ent, mk_z, roughness
- ~0.1 сек на ряд
- Идеально для первичного анализа больших датасетов

**Полный режим:**
- Вычисляет 17 расширенных статистик
- ~1 сек на ряд
- Для детального анализа

**Примеры использования:**

```bash
# Быстрый анализ малого датасета
python tests/test_latent_components.py --mode real \
    --tsf-path real_datasets/cif_2016_dataset.tsf --fast

# Анализ с ограничением числа рядов
python tests/test_latent_components.py --mode real \
    --tsf-path real_datasets/weather_dataset.tsf \
    --max-series 50 --fast

# Полный анализ (медленнее, но больше статистик)
python tests/test_latent_components.py --mode real \
    --tsf-path real_datasets/m4_monthly_dataset.tsf \
    --max-series 100
```

**Выходные данные:**
- `{category}_histograms.png` - Гистограммы 6 статистик с 95% доверительным интервалом
- `{category}_summary.json` - JSON отчет со сводными статистиками (mean, std, median, min, max, q25, q75)

**Визуализация с доверительным интервалом:**
- Данные обрезаются по 95% ДИ (2.5-й и 97.5-й перцентили) для удаления выбросов
- На графике указывается: число обработанных рядов, mean, std обрезанных данных
- Красная пунктирная линия - медиана
- Серые пунктиры - границы доверительного интервала

Генерирует 10 сэмплов для каждой компоненты, вычисляет статистики:

## Латентные компоненты

### 1. ARIMA (AutoRegressive Integrated Moving Average)

**Математика:**
```
Φ(B)(1 - B)^d y_t = c + Θ(B)ε_t
```
где `Φ(B)` - AR полином, `Θ(B)` - MA полином, `d` - порядок интегрирования

**Параметры:**
- `ar_params` - коэффициенты AR части
- `ma_params` - коэффициенты MA части
- `d` - порядок интегрирования (0, 1, 2)
- `intercept` - константа (смещение уровня)
- `sigma` - стандартное отклонение шума
- `burn_in` - длина начального участка для отбрасывания

**Характерное время генерации:** ~0.02 мс/шаг

---

### 2. KernelSynth (Гауссовские процессы)

**Математика:**
```
f(t) ~ GP(m(t), k(t, t'))
m(t) = a·t + b + c·exp(d·t)
```

**Ядра:**
- **RBF**: `k(t,t') = σ² · exp(-||t-t'||²/(2ℓ²))`
- **Periodic**: `k(t,t') = σ² · exp(-2sin²(π|t-t'|/p)/ℓ²)`
- **RQ**: `k(t,t') = σ² · (1 + ||t-t'||²/(2αℓ²))^(-α)`
- **Linear**: `k(t,t') = σ² · (t·t' + c)`

**Параметры:**
- `kernel_type` - тип ядра
- `lengthscale` - длина масштаба (ℓ)
- `variance` - дисперсия (σ²)
- `period` - период (для Periodic ядра)
- `alpha` - параметр (для RQ ядра)
- `mean_a, mean_b, mean_c, mean_d` - параметры функции среднего

**Характерное время генерации:** ~0.003 мс/шаг

---

### 3. TSI (Time Series Intrinsic)

**Математика:**
```
y(t) = Σ A_i · exp(-λ_i·t) · sin(2πf_i·t + φ_i)
```

**Параметры:**
- `frequencies` - частоты мод
- `amplitudes` - амплитуды мод
- `phases` - фазы мод
- `decays` - параметры затухания

**Характерное время генерации:** ~0.0004 мс/шаг

---

### 4. ETS (Exponential Smoothing)

**Математика:**
```
Уровень:  l_t = α·y_t + (1-α)·(l_{t-1} + b_{t-1})
Тренд:    b_t = β·(l_t - l_{t-1}) + (1-β)·b_{t-1}
Сезонность: s_t = γ·(y_t - l_{t-1} - b_{t-1}) + (1-γ)·s_{t-m}
```

**Параметры:**
- `model_type` - тип модели (ANN, AAN, AAA, MNN...)
- `alpha` - параметр сглаживания уровня
- `beta` - параметр сглаживания тренда
- `gamma` - параметр сглаживания сезонности
- `seasonality_period` - период сезонности
- `initial_level` - начальный уровень
- `initial_trend` - начальный тренд
- `initial_seasonal` - начальные сезонные компоненты

**Характерное время генерации:** ~0.003 мс/шаг

## Расширенные статистики

Модуль вычисляет **17 статистик** для каждой компоненты:

### Базовые статистики

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `mean` | Среднее значение | Центр распределения |
| `std` | Стандартное отклонение | Амплитуда колебаний |
| `min`, `max` | Мин/макс значения | Диапазон значений |
| `skewness` | Асимметрия распределения | 0 = симметрично, >0 = правосторонняя |
| `kurtosis` | Эксцесс (островершинность) | 0 = нормальное, >0 = тяжелые хвосты |

### Стационарность

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `adf_statistic` | ADF тест статистика | Более отрицательное = более стационарен |
| `adf_pvalue` | ADF тест p-value | <0.05 = стационарен (95% уверенность) |
| `is_stationary` | Флаг стационарности | True если p-value < 0.05 |

### Нормальность

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `normality_pvalue` | Тест Д'Агостино p-value | >0.05 = нет оснований отвергать нормальность |

### Автокорреляция

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `acf_lag1` | Автокорреляция лаг 1 | Близко к 1 = сильная память |
| `acf_lag5` | Автокорреляция лаг 5 | Долгосрочная зависимость |
| `acf_lag10` | Автокорреляция лаг 10 | Очень долгосрочная зависимость |

### Спектральные характеристики

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `spectral_entropy` | Энтропия спектра мощности | Меньше = более периодичен |
| `dominant_frequency` | Доминирующая частота | Основная частота колебаний |

### Сложность

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `sample_entropy` | Sample Entropy | Меньше = более предсказуем |
| `hurst_exponent` | Показатель Херста | 0.5 = случайное, >0.5 = тренд, <0.5 = возврат к среднему |

### Тренд

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `trend_slope` | Наклон линейного тренда | Скорость изменения во времени |
| `mann_kendall_tau` | Tau Кендалла | -1..1, сила монотонного тренда |
| `mann_kendall_pvalue` | p-value тренда | <0.05 = значимый тренд |

### Сезонность

| Статистика | Описание | Интерпретация |
|------------|----------|---------------|
| `seasonality_strength` | Сила сезонности | 0..1, ближе к 1 = сильнее |
| `detected_period` | Обнаруженный период | Длина периода в шагах |

## Настройка параметров латентных компонент

### Изменение параметров в коде

Для изменения параметров синтетических компонент отредактируйте `test_latent_components.py`:

```python
# ARIMA - настройка авторегрессии
arima_spec = ARIMASpec(
    type="arima",
    ar_params=torch.tensor([0.5, -0.3]),  # Коэффициенты AR
    ma_params=torch.tensor([0.4]),         # Коэффициенты MA
    d=1,                                    # Порядок интегрирования
    intercept=0.1,                          # Константа
    sigma=0.3,                              # Уровень шума
    burn_in=50                              # Прогрев
)

# KernelSynth - настройка гауссовского процесса
kernel_spec = KernelSynthSpec(
    type="kernel_synth",
    kernel_type="RBF",          # Тип ядра: RBF, Periodic, RQ, Linear
    lengthscale=0.3,            # Длина масштаба (ℓ)
    variance=1.5,               # Дисперсия (σ²)
    period=0.2,                 # Период (для Periodic)
    alpha=1.0,                  # Параметр (для RQ)
    mean_a=0.0,                 # Линейный тренд
    mean_b=0.0,                 # Сдвиг
    mean_c=0.0,                 # Экспонента
    mean_d=0.0                  # Показатель экспоненты
)

# TSI - настройка внутренних мод
tsi_spec = TSISpec(
    type="tsi",
    frequencies=[2.0, 5.0],     # Частоты мод
    amplitudes=[1.0, 0.5],      # Амплитуды мод
    phases=[0.0, 1.57],         # Фазы мод
    decays=[0.1, 0.2]           # Затухание мод
)

# ETS - настройка экспоненциального сглаживания
ets_spec = ETSSpec(
    type="ets",
    model_type="AAN",                    # Тип модели
    alpha=0.3,                           # Сглаживание уровня
    beta=0.1,                            # Сглаживание тренда
    gamma=0.0,                           # Сглаживание сезонности
    seasonality_period=12,               # Период сезонности
    initial_level=1.0,                   # Начальный уровень
    initial_trend=0.05,                  # Начальный тренд
    initial_seasonal=[0.1, 0.2, ...]     # Начальная сезонность
)
```

### Рекомендации по выбору параметров

**Для моделирования стационарных рядов:**
- ARIMA: `d=0`, `ar_params=[0.5~0.9]`
- KernelSynth: `kernel_type="RBF"`, `lengthscale=0.2~0.5`
- TSI: `decays=[0.5~2.0]` (быстрое затухание)

**Для моделирования трендов:**
- ARIMA: `d=1`, `intercept≠0`
- KernelSynth: `mean_a≠0` (линейный тренд)
- ETS: `model_type="AAN"`, `beta>0`

**Для моделирования сезонности:**
- KernelSynth: `kernel_type="Periodic"`, `period=T/сезонность`
- TSI: `frequencies=[1/период]`
- ETS: `model_type="AAA"`, `seasonality_period=T`, `gamma>0`

**Для моделирования нестационарности:**
- ARIMA: `d=1` или `d=2`
- KernelSynth: `mean_c≠0`, `mean_d>0` (экспоненциальный рост)
- ETS: `model_type` с трендом (AAN, MAN)
