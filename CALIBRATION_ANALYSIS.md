### 0.4. Детальные параметры многомерного анализа

**Размерность каналов (D):**

| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| **D_channels** | 4 | Базовая размерность для всех многомерных метрик |
| **T_fixed** | 100 | Фиксированная длина временного ряда |
| **Формирование батчей** | Последовательные ряды | 4 ряда подряд из общего списка |
| **Глобальное перемешивание** | ❌ Нет | Ряды не перемешиваются между датасетами |

**Почему D=4?**

1. **Совместимость с генератором:** Генератор создаёт данные с dim_range=(1, 3), что даёт D = dim + 1 = 2..4
2. **Вычислительная эффективность:** D=4 позволяет быстро вычислять сигнатуры (depth=2 → K=21 компонент)
3. **Недостаточно для `analyze_all_5d_combinations`:** Эта функция требует D ≥ 5

**Как формируются батчи:**

```python
# 1. Загружаем n_series=100 рядов из КАЖДОГО датасета
for ds_name in dataset_config:
    series = load_monash(ds_name, n_series=100)
    all_series.extend(series)

# 2. Всего рядов: ~24 датасета × 100 = ~2400 рядов
# 3. Обрезаем до T=100
series_mv = [s[:100] for s in all_series if len(s) >= 100]

# 4. Формируем батчи по 4 канала (без перемешивания!)
D_channels = 4
B = len(series_mv) // 4  # ~600 батчей
B = min(B, max_batches)  # max_batches=100 для "all"

# 5. Каждый батч — 4 ПОСЛЕДОВАТЕЛЬНЫХ ряда
batch_b = [series_mv[b*4], series_mv[b*4+1], ...]
mv_data.shape = (100, 100, 4)  # (B, T, D)
```

**Важное замечание:** Каналы в одном батче могут быть из одного датасета (например, 4 ряда из fred_md), так как глобальное перемешивание не применяется.

**Функция `analyze_all_5d_combinations`:**

- **Назначение:** Глубокий анализ высокоразмерных данных (D > 5)
- **Принцип:** Перебор всех сочетаний из 5 каналов: C(D, 5)
- **Примеры:**
  - D=5 → 1 комбинация
  - D=8 → 56 комбинаций
  - D=10 → 252 комбинации
- **Время:** В 240 раз медленнее `analyze_batch` для D=10
- **Текущее использование:** НЕ используется (D=4 < 5)

## 5. Рекомендации по калибровке

### 5.1. Приоритет 1: Архитектурные изменения для cross_corr

**Проблема:** Увеличение `l_range` не помогает — требуется общая латентная структура.

**Решение:** Реализовать shared latent factors в latent.py:

```python
# Вместо независимых z[:, k] для каждого канала:
z_common = z[:, :2]  # Общие факторы для всех каналов
z_specific = z[:, 2:]  # Специфичные для каналов

# x[:, 0] = f(z_common, z_specific[:, 0:2])
# x[:, 1] = g(z_common, z_specific[:, 2:4])
# x[:, 2] = h(z_common, z_specific[:, 4:6])
```

### 5.2. Приоритет 2: Сбалансировать сложность графа

**Проблема:** max_depth=10 даёт plateau или ухудшение entropy.

**Решение:** Уменьшить до оптимального значения:

```python
transform_prior = TransformationPrior(
    max_depth=8,  # Было 10 (слишком глубоко)
    node_kind_probs={"op": 0.55, "terminal": 0.30, "constant": 0.15},
    op_weights={
        "mul": 0.50,  # Умеренное значение
        "sin": 0.60,
        "cos": 0.40,
        "tanh": 0.40,
    },
)
```

### 5.3. Приоритет 3: Добавить нестационарность

**Проблема:** adf_ct показывает избыточную стационарность.

**Решение:** Увеличить долю нестационарных процессов:

```python
latent_prior = LatentPrior(
    type_probs={"arima": 0.15, "kernel_synth": 0.45, "tsi": 0.15, "ets": 0.25},
    kernel_synth=KernelSynthPrior(
        mean_a_range=Range(min=-2.0, max=2.0),  # Усилить тренды
        mean_c_range=Range(min=-0.5, max=0.5),  # Экспоненциальные тренды
    ),
)
```

### 5.4. Приоритет 4: Увеличить разнообразие частот

**Проблема:** fft_mean показывает недостаток высокочастотных компонент.

**Решение:** Расширить диапазон частот TSI:

```python
tsi=TSIPrior(
    frequency_range=Range(min=0.2, max=15.0),  # Было [0.5, 8.0]
    n_modes_range=(5, 10),  # Увеличить число мод
)
```
